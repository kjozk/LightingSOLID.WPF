# LightingSOLID.WPF

## 概要
LightingSOLID.WPF は、照明器具を題材にオブジェクト指向プログラミング (OOP) を学ぶための WPF アプリケーションです。このプロジェクトでは、実際の照明器具の特性をモデル化し、設計の中で GOF (Gang of Four) デザインパターンや SOLID 原則を意識した構造を採用しています。

---

## 特徴
### 1. **オブジェクト指向プログラミング (OOP) の学習**
- 照明器具を抽象化し、インターフェースや抽象クラスを活用して多様な照明器具をモデル化。
- 継承やポリモーフィズムを通じて、コードの再利用性と拡張性を実現。
- 実際の照明器具の特性（明るさ、消費電力、色温度など）をプログラムで表現。

### 2. **GOF デザインパターンの適用**
- **Template Method パターン**:
  - `NonDimmableLightingDevice` クラスを基底クラスとして、HID 照明器具や LED 照明器具の共通動作をテンプレート化。
- **Factory Method パターン**:
  - `LedLightingDeviceFactory` を使用して、特定の LED 照明器具（昼光色、昼白色、電球色など）を簡単に生成。
- **Strategy パターン**:
  - 調光可能なデバイス (`DimmableLedDevice`) において、明るさの増減ロジックを柔軟に切り替え可能。

### 3. **SOLID 原則に基づく設計**
- **単一責任の原則 (SRP)**:
  - 各クラスは単一の責任を持ち、UI ロジック (`LightingDeviceViewModel`) とビジネスロジック (`ILightingDevice`) を明確に分離。
- **開放/閉鎖の原則 (OCP)**:
  - 新しい照明器具を追加する際、既存のコードを変更せずに拡張可能。
- **リスコフの置換原則 (LSP)**:
  - `ILightingDevice` インターフェースを実装するすべてのクラスは、置換可能な形で動作。
- **インターフェース分離の原則 (ISP)**:
  - `IDimmable` インターフェースを分離し、調光機能を持たないデバイスに不要な依存を排除。
- **依存性逆転の原則 (DIP)**:
  - 高レベルモジュール（UI ロジック）は、低レベルモジュール（具体的な照明器具クラス）に依存せず、抽象化されたインターフェースに依存。

---

## 主なクラスと構造
### **1. インターフェース**
- `ILightingDevice`:
  - 照明器具の基本的な操作（点灯、消灯、明るさ、消費電力など）を定義。
- `IDimmable`:
  - 調光可能なデバイス専用の操作（明るさの増減）を定義。

### **2. 抽象クラス**
- `BasicLightingDevice`:
  - 調光機能を持たない照明器具の基底クラス。
  - HID 照明器具や LED 照明器具の共通動作を実装。
- `DimmableLedDevice`:
  - 調光可能な LED 照明器具を表現。

### **3. 具象クラス**
- `MetalHalideLightingDevice`:
  - メタルハライドを表す。

## 学習ポイント
1. **OOP の基本概念**:
   - 抽象化、カプセル化、継承、ポリモーフィズムを実際のコードで体験。
2. **設計パターンの実践**:
   - GOF デザインパターンを適用した設計を学習。
3. **SOLID 原則の適用**:
   - 保守性と拡張性を意識した設計の重要性を理解。
